Openfire Social Plugin Readme

Requirements

You need Openfire 4.0 and above with Java 1.8.
You will need to have configured Openfire with MySQL or compatible DB. Other DBs have not beed tested or confirm suitable for WordPress.

Overview

Openfire Social is a plugin which combines WordPress and BuddyPress to provide a fully collaborative online social community.

Openfire is auto-configured as follows when plugin is installed and reset back when plugin is uninstalled:

  • All Openfire users are WordPress users and Openfire admin console cannot add or modify users
  • All Openfire user groups are read from BuddyPress groups Openfire admin console cannot add or modify user groups
  • Group Chat rooms and Roster groups are auto-created every time a BuddyPress group is created
  • User Rosters are updated every time a BuddyPress friendship is made or broken.

Please note that Only Openfire and MySQL required. No Apache, No PHP. The Openfire Social plugin has embedded Quercus PHP Engine which is a 100% Java implementation of PHP 5.

Quercus presents a new mixed Java/PHP approach to web applications and services where Java and PHP tightly integrate with each other. WordPress and BuddyPress interface directly with Openfire. This capability is made possible because 1) PHP code is interpreted/compiled into Java and 2) Quercus and its libraries are written entirely in Java.

How to use

WordPress admin web page - http://yourserver:7070/ofsocial/wp-admin

WordPress social home page - http://yourserver:7070/ofsocial/wp-user

Communications toolbar at bottom of screen

  • Single/Auto sign-in to Openfire from WordPress
  • Roster of BuddyPress Friendships showing Presence
  • IM chat with BuddyPress friends
  • MUC chats with BuddyPress group members
  • Openfire Meetings audio/video with friends
  • Openfire Meetings audio/video with a Buddypress group
